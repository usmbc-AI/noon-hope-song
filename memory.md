# 🎵 정오의 희망곡 - 자동 선곡 프로그램 (memory.md)

> 그날의 무드(계절·날짜·날씨 등)에 어울리는 노래를 자동으로 추천하는 프로그램.
> 이 파일은 프로젝트가 진행되며 계속 업데이트되는 "기억 노트"입니다.

_최초 작성: 2026-07-09_

---

## 1. 프로젝트 개요
- **목표**: 방송 당일의 분위기에 맞는 곡을 자동으로 선곡해주는 도구 제작
- **무드 판단 입력값(초안)**: 계절 / 날짜(요일·기념일·시즌) / 날씨 / (그 외 추가 예정)
- **모티브 프로그램**: MBC FM4U <정오의 희망곡> — 매일 낮 12~14시 방송하는 장수 음악 프로그램

## 2. 참고: 실제 <정오의 희망곡> 톤 (리서치 기반)
- 따뜻하고 희망적인 **한낮(점심시간)의 무드**가 핵심 정체성
- 폭넓은 청취층(직장인·주부 등)을 대상으로 함
- 가요·발라드·K-pop·팝·추억의 명곡까지 **장르/시대를 넓게** 아우름
- 요일마다 코너·테마가 있음 (참고용, 우리 프로그램에 반영할지는 선택)

## 3. 확정된 사항
- **음악 소스 (A-3)**: AI가 곡명·가수를 추천만 함 (실제 재생 없이 **선곡표 용도**)
- **자동화 (B-1)**: **완전 자동** — 오늘 날짜·날씨를 스스로 읽어 선곡
- **출력 (C-2)**: 곡 정보 + **DJ 멘트/추천 이유(코멘트)** 함께 제공
- **선곡 범위 (D)**: **국내 가요 위주**
- **날씨 기준 지역 (F-2)**: **울산 고정** (위도 35.54, 경도 129.31)
- **곡 수 (C-1)**: **8~10곡**
- **무드 로직 (E-2)**: AI가 그날 상황(계절·요일·공휴일·날씨)을 해석해 무드 결정
- **실행 환경 (F-1)**: AI 기반 웹앱 (버튼 클릭 → 자동 선곡). 날씨는 Open-Meteo 자동 조회,
  실패 시 수동 선택으로 폴백. 선곡·멘트는 Claude API로 생성

## 4. 열린 질문들 (킥오프)

### A. 음악 소스 🎧 **(가장 중요 / 먼저 결정 필요)**
곡 후보가 어디서 오는지가 전체 구조를 좌우합니다.
- [ ] A-1. 내가 직접 준비한 곡 목록(CSV/엑셀 등)에서 고른다
- [ ] A-2. 스트리밍/음원 API(멜론·스포티파이·유튜브 등)에서 실시간으로 가져온다
- [ ] A-3. 소스 없이 AI가 곡명·가수를 추천만 한다 (실제 재생 없음, 선곡 리스트 용도)

### B. 자동화 수준 ⚙️
- [ ] B-1. 완전 자동 — 오늘 날짜·날씨를 스스로 읽어서 선곡
- [ ] B-2. 반자동 — 담당자가 몇 가지(예: 오늘 콘셉트, 날씨)를 입력하면 선곡

### C. 출력 형태 📝
- [ ] C-1. 몇 곡을 뽑을지 (예: 하루 5곡 / 20곡 / 방송 2시간 분량 등)
- [ ] C-2. 곡 정보만? 아니면 DJ 멘트/추천 이유(코멘트)도 함께?
- [ ] C-3. 결과물 형태 (화면 표시 / 텍스트 / CSV·엑셀 / 선곡표 문서 등)

### D. 음악 범위 🌏
- [ ] D-1. 국내 가요 위주? 팝·해외곡 포함? 비율은?
- [ ] D-2. 시대 범위 (최신곡 위주 / 추억의 명곡 포함 / 무관)
- [ ] D-3. 피하고 싶은 장르·분위기 있는지 (예: 너무 격한 곡 제외 등)

### E. 무드 판단 로직 🧠
- [ ] E-1. 규칙 기반 매핑(예: "비 오는 날 → 잔잔한 발라드")을 직접 정해둘지
- [ ] E-2. AI가 그날 상황을 해석해 알아서 무드를 잡게 할지
- [ ] E-3. 날짜에서 고려할 요소 (요일 무드 / 공휴일·기념일 / 절기·시즌 등)

### F. 실행 환경 💻
- [ ] F-1. 어떤 형태로 쓰고 싶은지 (파이썬 스크립트 / 웹앱·버튼 클릭 / 문서 산출물 등)
- [ ] F-2. 날씨·위치 데이터 (특정 지역 고정? 날씨 API 사용 가능 여부?)

## 6. 기술 설계 메모
- **형태**: 단일 HTML 웹앱 (버튼 클릭 → 자동 선곡)
- **날짜/계절/절기**: JS 자동 계산. 계절(봄3-5·여름6-8·가을9-11·겨울12-2),
  주요 공휴일, **24절기(근사 날짜+의미)** → 오늘의 절기를 카드/선곡에 반영
- **날씨**: Open-Meteo로 울산(35.54,129.31) 실시간 조회 → 실패 시 수동 선택 폴백.
  하늘상태+**기온·습도·바람**을 선곡/오프닝 멘트/곡 이유에 활용
- **선곡 2-경로**:
  1) **AI 모드**: Claude API 호출로 JSON 생성 (배포 환경에 API 브리지 있을 때)
  2) **내장 엔진(폴백)**: API 연결 실패 시 자동 전환. 실제 국내곡 라이브러리(약 37곡,
     계절·무드 태그)에서 오늘 무드로 스코어링해 9곡 선정(인디 3곡 보장),
     날씨/절기 반영한 멘트·코멘트 템플릿 생성. 날짜 시드로 매일 변주
- **선곡 구성**: 국내 가요 위주 8~10곡 + **인디 2~3곡("숨은 명곡")**, 🎧 배지 표시
- **정확성 검증**: 클라이언트에서 **애플 뮤직(iTunes Search, JSONP)** 발매 대조 →
  ✓(확인)/?(확인 안 됨)/–(조회 실패) 진행형 표시, 헤더 "발매 확인 n/총"
- **주의**: 인디 곡은 DB 미등록으로 ?가 뜰 수 있음 → 방송 전 음원/저작권 최종 확인

## 7. 트러블슈팅 로그
- (2026-07-09) 선곡 시 "Failed to fetch" 발생.
  - 1차 가설: web_search 서버 도구 미지원 → 제거했으나 동일 증상 지속.
  - 최종 진단: **현재 미리보기 환경에 Claude API 브리지(프록시)가 없어** 브라우저가
    api.anthropic.com에 직접 접근 못 함(CORS/네트워크 차단). 날씨(Open-Meteo)는 공개 API라 정상.
  - 해결: **내장 선곡 엔진을 폴백으로 추가** → 어떤 환경에서도 곡이 나오도록. AI 모드는
    API 브리지가 있는 배포 환경에서 자동 사용됨. (원하면 사용자 API 키 입력 방식도 추가 가능)
- (2026-07-09) 내장 엔진 코멘트가 곡 분위기와 어긋나는 문제(예: 크러쉬 "Beautiful"은 잔잔한데
  '경쾌한 리듬'으로 표기). 원인: 곡의 무드 태그(v)가 코멘트를 결정하는데 태그가 부정확.
  → 크러쉬 Beautiful [신남→설렘/감성], 혁오 TOMBOY [잔잔 제거], 새소년 난춘 [청량 제거] 보정.
  ※ 라이브러리 확장 시 각 곡의 v(무드) 태그를 실제 곡 템포/분위기에 맞게 검수할 것.
